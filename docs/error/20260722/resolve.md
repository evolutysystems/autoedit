# 長尺動画でテロップだけが徐々にズレる不具合 調査・回答設計書

対象エラー: `docs/error/20260722/error.md`
主対象ソース: `src/modules/subtitle_generator.py`（`WhisperTextSource`／`adjust_display_timing`／`burn_subtitle`）
関連ソース: `src/modules/silence_cutter.py`（`extract_segment`／`_concat_demux`）／`src/modules/ffmpeg_runner.py`（`build_encode_options`）
前提: 直近 `docs/error/20260719/resolve.md` の対策A（焼き込みの `setpts=N/FRAME_RATE/TB` → `fps={output_fps}` 実時刻CFR化）適用済み。
本書は `docs/CLAUDE.md` 方針に準拠した**調査・回答設計書**であり、実装は承認後に行う（推測実装はせず、実測が要る点は §6・§7 に明記する）。

---

## 1. 事象と切り分け（最重要）

- 前半のズレは 20260719 対策で解消した。
- **映像と音声はズレていない**（＝A/V 同期は取れている）。
- **テロップだけが徐々にズレる**。30分動画で **8分経過あたりから顕在化**し、時間とともに拡大する。

この切り分けが決定的である。20260719 の事象は「**映像 vs (音声＋テロップ)**」がズレるものだった（テロップは音声と一致）。今回はその逆で「**(映像＋音声) vs テロップ**」がズレる。

> **結論の核**：映像と音声が一致しているなら、コンテナ側のPTS・DTS・Timebase・連結境界は**すでに正しく揃っている**。テロップ（ASS）は絶対時刻ラベルで焼き込まれるため、A/V が合っているのにテロップだけが経過時間に比例してズレるということは、**ズレの発生源はコンテナ(FFmpeg)層ではなく、テロップの時刻そのものを作っている音声認識(Whisper)のタイムスタンプ側**にある可能性が極めて高い。

したがって error.md に列挙された6つの対策（`-ss` 位置／再エンコード回数／`setpts`・`asetpts`／concat demuxer／`-copyts`／`-avoid_negative_ts`）は**すべて A/V 同期＝コンテナ層の対策**であり、**今回のテロップ・ドリフトの主因には当たらない**というのが本書の一次結論である（§2 で1件ずつ検証）。

要約：**6件中4件は既に実装済みか対処済み、残る2件（-copyts / 出力後 -ss）は今回の症状に無効または非推奨**。いずれも「映像と音声が既に一致している」今回の切り分けを動かせない。以下、要点のみ補足する。

### 2-1. `-ss` の位置（対策1）
現行 `extract_segment` は `-ss` を `-i` の**前**に置く（高速シーク）。error.md の「悪い例」はこれに一致するが、**それが不正確になるのは `-c copy`（ストリームコピー）時**である。本コードは各区間を**再エンコード**するため、modern FFmpeg は入力前 `-ss` でも「直前キーフレームまで高速シーク→目標時刻までデコード破棄」を行い、**フレーム精度でカットされる**（`silence_cutter.py:173-174` のコメントもこの前提）。
なお `-to`（基準が曖昧）は避け `-t`（区間長）を使用しており（`:188`）、これも妥当。
**「入力後 -ss（＝正確版）」への変更は、区間ごとにフレーム0からの全デコードを強制し、`docs/error/20260611`・`20260719 §3 B-3` で棄却済みの低速化/ハングを招く。今回の症状にも無関係なので採用しない。**

### 2-2. 多重エンコード（対策2）
現行の世代数は「区間抽出（再エンコード）→ 連結（`-c copy`）→ 焼き込み（再エンコード）→ OP/ED（該当時のみ再エンコード）」の**2〜3世代**であり、error.md が懸念する5世代ではない。各再エンコードは `-fps_mode cfr -r {fps}` と `-fflags +genpts` で**PTS/DTS/Timebase を毎回クリーンに再生成**するため、世代を跨いだTS劣化の累積は起きにくい。
「理想＝全編集をFilterで1回エンコード」は `extract_mode="filter"`（`trim`＋`concat` 単一パス）に相当するが、長尺で全デコード/巨大split となりハングするため **20260719 §3 B-3 で棄却済み**。テロップは焼き込みの**1世代のみ**で載る（以降はピクセル搬送）ので、多重エンコードはテロップ・ドリフトの原因にならない。

### 2-3. `setpts`/`asetpts`（対策3）
20260719 対策A で焼き込みの `setpts=N/FRAME_RATE/TB` は**廃止**され、映像は `fps={fps}`（実PTS基準CFR化）、音声は `-af aresample=async=1:first_pts=0` に置換済み（`subtitle_generator.py:612,622`）。フィルタ経路（`cut_and_concat`）では `setpts`/`asetpts` を対で使用済み（`:106,117`）。**未対の setpts は現行経路に存在しない**ため、この指摘は既に解消されている。

### 2-4. concat demuxer（対策4）
`_concat_demux` は `-f concat -safe 0 -i list.txt` の**concat demuxer（ファイルリスト方式）を使用**しており、error.md の推奨と完全一致（`silence_cutter.py:353-357`）。`concat:` プロトコルは未使用。**むしろ**この demuxer × `-c copy` が 20260719 で特定した「区間境界の約21ms空白」を生むが、それは A/V 同期の問題で**20260719 対策で解決済み**であり、テロップ時刻とは別軸。

### 2-5. `-copyts`（対策5）
未使用。本アーキテクチャは区間ごとに `-avoid_negative_ts make_zero` でTSをゼロ起点へ正規化し、焼き込みで `fps` により実時刻CFRへ組み直す設計である。ここに `-copyts`（ソースの絶対TSを保持）を混ぜると、**区間抽出・連結で意図的にリセットしているTSと衝突**し、負PTS/不連続を再導入しかねない。今回の「テロップだけズレる（A/Vは合っている）」症状に対しては**効果が期待できず、副作用リスクが高い**ため非推奨。

### 2-6. `-avoid_negative_ts make_zero`（対策6）
`extract_segment`（`:209`）と `_concat_demux`（`:365`）で**既に付与済み**。これは連結境界の負PTS・mux不整合の是正であって、A/V が既に合っている今回の症状には効かない。

## 3. 真因の考察（テロップ時刻の生成源＝Whisper）

A/V が一致してテロップだけが**経過時間に比例して**ズレる以上、疑うべきは ASS へ書き込む時刻を生む音声認識である。テロップの時刻は次の2段でしか決まらない。

1. **Whisper のタイムスタンプ**（`WhisperTextSource._recognize` → `_speech_bounds`、`subtitle_generator.py:343-373`）
2. **表示タイミング整形**（`adjust_display_timing`、`:746-775`）— 各テロップの start は Whisper の start をそのまま採用し、end のみ「次発話 or 発話末+max_hold」で決める。**start（ズレの見え方を支配する量）は Whisper 由来そのまま**。

したがってテロップ start のドリフト＝**Whisper タイムスタンプのドリフト**とほぼ等価。以下が有力な機序（要実測）。

### 3-1. 【第1候補】無音カット後の「隙間の無い連続発話」で Whisper の時刻が累積ドリフトする
- テロップ工程は**無音カット後の `silence_cut.mp4`** を入力に Whisper を回す（`subtitle_generator.py:897,902`）。この音声は**間(ま)が全て除去され、ほぼ切れ目なく発話が連続**している。
- Whisper は内部30秒窓でデコードし窓を跨いで時刻を継ぐ。**無音アンカーが乏しい長い連続発話ほど、窓あたりの微小誤差が再アンカーされずに累積**する。`word_timestamps=True`（DTWアライメント、`:320,386`）は緩和するが完全ではない。
- これは「**冒頭は一致→経過とともに拡大→8分あたりで体感**」という今回の症状の型と整合する。A/V コンテナとは独立に起きるため「映像と音声は合うのにテロップだけズレる」も満たす。

### 3-2. 【第2候補】VAD の時刻マッピング
- `whisper_vad_filter=True`（既定、`:307,388-391`）。VAD は非発話区間を除去して認識し、**区間境界の尺で時刻を元タイムラインへ写像し直す**。faster-whisper のバージョンによっては、長尺でこの写像に系統的な偏りが乗ることがある。
- 無音カット済み音声に更に VAD をかける二重ゲート構成であり、写像誤差の温床になり得る。

### 3-3. 【第3候補・低】焼き込み音声の `aresample=async=1`
- 焼き込みは `-af aresample=async=1:first_pts=0`（`:622`）。Whisper が見た音声（焼き込み前）と、`async=1` 通過後の音声で時間軸が微妙にズレると、テロップが**焼き込み後の音声**に対してドリフトし得る。
- ただし `silence_cut.mp4` の音声は連続で `async=1` は実質ほぼ無補正のはずで、優先度は低い。ゼロではないため測定対象に含める。

いずれも**動画（音声波形とASS時刻の突き合わせ）が無いと確定できない**。CLAUDE.md 方針に従い、ここでは断定せず §4 の実測で切り分ける。

## 4. 検証計画（30分実素材で切り分ける）

Aが本命、B/C は候補潰し。**Aだけで機序が確定する見込み**。

- **A. Whisper 時刻 vs 実発話の実測**：`silence_cut.mp4` を Whisper に通し、生成 ASS の各 start と、`silence_cut.mp4` の**実際の発話立ち上がり**（`silencedetect` 高閾値 or 手動）を突き合わせ、経過時間に対するズレ量をプロットする。**8分付近でズレが立ち上がるか／単調増加か**を確認（20260719 verify と同じ流儀）。
- **B. VAD 有無比較**：`whisper_vad_filter` を True/False で同一素材に適用し、ドリフト曲線を比較。VAD 写像が主因なら False で大幅改善するはず。
- **C. 連続発話の影響切り分け**：**無音カット前の原動画**を Whisper に通した ASS と、**無音カット後**の ASS のドリフトを比較。後者だけ悪化するなら §3-1（連続発話でのアンカー欠如）が主因。
- **D. 焼き込み音声の影響**：焼き込み後の音声を書き出し、Whisper 再認識の時刻と ASS 時刻を比較（§3-3 の確認）。
- 併せて `condition_on_previous_text`（現行 False, `:312`）、`beam_size`（現行8, `:298`）、`whisper_model` の違いがドリフトに与える影響も A の枠内で確認する。

## 5. 修正方針候補（実測で機序確定後に選択・本書では実装しない）

機序に応じて以下から選ぶ。**いずれも `setting.json` 管理・既存破壊なし・不要ライブラリ追加なしを満たす範囲**で検討する。

- **§3-1/§3-2 が主因の場合**
  - **(a) 休眠中の発話ゲートの活用**：`_gate_with_settings`／`gate_timeline_by_speech`（`:669-736`）は実装済みだが `run()` で**コメントアウトされ休眠中**（`:912-919`）。テロップ start を、Whisper 時刻ではなく**`silencedetect` で得た実発話立ち上がり**へスナップ（`mode="trim"`）すれば、Whisper のドリフトを実音声基準で再アンカーでき、累積を断てる可能性がある。**FFmpeg 側の確実な時刻で ASR の時刻を矯正する**という筋であり、今回の症状に最も直接的。
  - **(b) VAD パラメータ調整/無効化**：`whisper_vad_filter` を設定で切替可能な既存キーで検証（`:307`）。ただしハルシネーション抑制(20260626 対策A)を弱める副作用があるためトレードオフを明記。
  - **(c) 長時間連続発話の分割認識**：Whisper 入力を無音カット前音声にする、または一定長で区切って窓ドリフトを抑える案（アーキテクチャ変更を伴うため優先度は実測次第）。
- **§3-3 が主因の場合**：焼き込みの音声処理（`aresample` パラメータ）を見直す。ただし 20260719 で導入した同期維持の役割があるため、A/V 回帰がないことを最優先で確認する。

## 6. 既存対策との整合（回帰防止）

- 20260719 対策A（焼き込みの `fps` 実時刻CFR化・`+genpts`・`aresample`・`-fps_mode cfr -r`）は**維持**する。テロップ側の矯正で A/V 同期を壊さないことを必須確認事項とする。
- 20260630 のパスエスケープ（2段階）・`extract_mode="seek"` 経路（20260610/20260611 のハング対策）も**変更しない**。
- 20260626 対策A（VAD/`no_speech`/`condition_on_previous_text` によるハルシネーション抑制）と、20260627 対策A/B（`word_timestamps` と `adjust_display_timing`）は今回の調整対象と隣接するため、変更時は当時の目的（幻聴字幕抑制・表示タイミング）を退行させないこと。

## 7. 不明点・確認事項（レビュー／実機確認対象）
