# エラー解決報告 (resolve)

## 1. 概要
| 項目 | 内容 |
| --- | --- |
| 発生日 | 2026-06-02 |
| 例外種別 | `TypeError: not enough arguments for format string` |
| 発生箇所 | `src/modules/concat_processor.py` 82行目 |
| 処理工程 | オープニング・エンディング結合 (`concat_processor.run`) |
| 影響 | ログ出力時に例外が発生し、パイプラインが中断 |

## 2. 発生コード
```python
# src/modules/concat_processor.py:82
_logger.info("%s 開始: %s + %s", label, *(parts,))
```
- `label` … ログラベル文字列 (例: `'オープニング結合'`)
- `parts` … 結合対象パスのリスト (要素2個)
  例: `['D:/StreamPipeline/dev/assets/op.mp4', '...subtitle.mp4']`

## 3. 原因
書式文字列 `"%s 開始: %s + %s"` は **`%s` を3個** 要求している。

しかし引数展開部 `*(parts,)` は

1. `(parts,)` … 「`parts` リストを **1要素だけ持つタプル**」を生成
2. `*` でそのタプルを展開 → `parts` (リスト1個) が **1引数** として渡る

結果、ロガーへ渡る引数は `label` と `parts` の **2個** のみとなり、
書式の `%s` 3個に対して引数が1個足りず `TypeError` が発生した。

つまり「リストの中身2要素を展開する」つもりが、
「リストそのものを1個渡す」コードになっていたことが直接原因。

## 4. 修正内容
`*(parts,)` を `*parts` に修正し、リストの2要素を個別引数として展開する。

```python
# 修正前
_logger.info("%s 開始: %s + %s", label, *(parts,))

# 修正後 (parts の2要素を展開して %s 3個に対応)
_logger.info("%s 開始: %s + %s", label, *parts)
```

### 引数対応 (修正後)
| 書式 | 引数 |
| --- | --- |
| 1個目 `%s` | `label` |
| 2個目 `%s` | `parts[0]` (結合元1: 素材 or 本編) |
| 3個目 `%s` | `parts[1]` (結合元2: 本編 or 素材) |

## 5. 修正対象ファイル
- `src/modules/concat_processor.py` (82行目)

## 6. 再発防止
- ロガーの遅延フォーマット (`_logger.info(fmt, *args)`) では、
  `%s` の個数と引数の個数が一致することを確認する。
- リストを展開する際は `*(list,)` ではなく `*list` を用いる。
  - `*(x,)` は「x を1個含むタプルの展開」= `x` 1個渡し
  - `*x` は「シーケンス x の各要素の展開」
- 同種の `_logger` 呼び出しが他工程 (無音カット・テロップ生成) に
  ないか併せて確認することを推奨。

## 7. 確認結果
- 修正により書式 `%s`×3 と引数3個 (`label` + `parts` 2要素) が一致し、
  `TypeError` は解消される見込み。
- 結合処理本体 (`concat(...)`) のロジックには影響なし
  (ログ出力行のみの修正のため既存実装を破壊しない)。
