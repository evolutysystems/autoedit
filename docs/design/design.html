1. システム概要

入力動画 1 本に対し、無音カット → フルテロップ生成 → OP/ED 結合 → 出力 を 1 クリックで自動実行する。

OS

Windows 11

Python

3.14

GUI

PySide6

外部ツール

FFmpeg

PipelineRunner
 ├─ uses → SilenceCutter
 ├─ uses → SubtitleGenerator
 ├─ uses → ConcatProcessor
 ├─ uses → OutputWriter
 ├─ uses → FFmpegRunner
 └─ holds → PipelineContext

PipelineContext
 ├─ input_path : str
 ├─ working_dir : str
 ├─ intermediate_paths : dict
 ├─ output_path : str
 ├─ settings : dict
 └─ progress_callback : Callable

SubtitleGenerator
 ├─ TextSource (抽象)
 │   └─ WhisperTextSource (faster-whisper 音声認識)
 ├─ FontProfile (フォント差替吸収)
 └─ run(context) -> str
       音声認識 → ASS 生成 → ASS 焼き込み

7. エラー処理方針

  • 独自例外階層 AutoEditError 配下に ConfigError / FFmpegError / InputError / SubtitleError を定義
  • FFmpeg は exit code != 0 で例外化、コマンドと stderr 末尾をログ
  • OP / ED 未設定は例外でなく スキップ
  • 失敗時、中間ファイルは保持しデバッグ可能に

8. ログ出力方針

  • Python 標準 logging のみ使用 (追加依存なし)
  • コンソール + logs/autoedit_YYYYMMDD.log
  • レベルは setting.json で切替 (デフォルト INFO)
  • FFmpeg コマンド全文は DEBUG レベル

9. 将来拡張方針

  • フォント変更: FontProfile 経由で差替可能
  • 音声認識差替: 既定 WhisperTextSource (faster-whisper)。TextSource 抽象経由で Vosk 等へ差替 (subtitle.engine で選択)
  • 配信スタイル分岐: broadcast_style を参照する Strategy
  • 複数ファイル一括: run_pipeline を for ループで呼ぶ薄いラッパ
  • クラウド出力: OutputWriter 抽象化
